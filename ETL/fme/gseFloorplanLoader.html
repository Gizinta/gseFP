gseFloorplanLoader

This process creates Flooplan data from one or more source CAD Drawings.

There are 3 datasets that are updated in this process as shown below

The fourth dataset for the Building outline is created first with the gseBuildingOutlineLoader.fmw process. The Floorplan load process relies on both the BuildingOutline feature and a corresponding row in the Active_Floors table.

The datasets are basically copied from the source drawings, the main difference is that the number of floorplan line features is reduced by aggregating them by source CAD layer. This greatly reduces the number of features in the database and improves drawing performance significantly.

The 2 main other functions performed in this process are to re-project the data from paper space to Web Mercator coordinates, and also to fix any 'catwalks' in the CAD drawings. Typically the room-within-room situations on a floor involve drawing lines very close to together in CAD but we want to erase those lines for GIS features and have polygons within polygons.

  • InteriorSpace - these are the space polygons. The source must be closed polygons with at least a room number attribute.
  • Floor_Poly - The base floorplate area as created in CAD corresponding to your rules that define floor area. This can be include more than one feature per floor.
  • FloorplanLine - Aggregated floorplan lines by layer in CAD. This typically results in several to a dozen features per floor.

Once these features are created successfully, the FloorplanDerive process is the next step.
